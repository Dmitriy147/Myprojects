Панель управления и мониторинга СУДОС Цирконий-М

Клиент-сервер для отображения состояния служб на серверах СУДОС, а так же оркестрация управляющих команд для серверов и АРМов СУДОС.

О проекте

При переводе СКУД объекта под управление ОС АстраЛинукс возникла потребность в упращении администрирования СУДОС администраторами, привыкшими к работе в среде Windows. Так же в критических ситуациях, при восстановлении функционирования СКУД дежурным персоналом, имеется вероятность некорректного вмешательства в работу СКУД. Поэтому, для прозрачности текущего состояния системы и для блокировки некорректных действий персонала, была создана данная программа.

Демонстрация основных возможностей проекта

Основные функции

Кроссплатформененость

Возможность разворачивания программы на хостах с ОС Windows/AstraLinux1.7, а так же используя технологию Docker.

Максимальная информативность

Статус состояния служб i10, Kernel, Cluster, наличие пинга сервера.

Оркестрация функций управления

В зависимости от состояния служб серверов активируются/деактивируются кнопки доступного управления серверами.

Статистика памяти/дисков/ЦПУ по серверам

Выводв текущего состояния используемой ЦПУ/жёсткого диска/памяти агентами на серверах.

Резервное копирование БД

Возможность резервного копирования БД СУДОС с любого из серверов, а так же, при остановке работы СУДОС, восстановление БД из любой из имеющихся слепков.

Удалённый перезапуск хостов

Удалённый Reboot хостов и серверов из списка по ssh-команде.

Гибкое логирование

Настраиваемое логирование на стороне серверной и клиентской части. Ротация по-суточная. Ограничение максимального числа логов.

Используемые технологии

Golang Fyne Gin Websocket Zerolog Cron-go HTML JavaScript Tailwind CSS

Серверная часть, а так же агенты на двух серверах запущены в виде демонов. В режиме ожидания агенты на серверах ожидают подключения серверной части программы. Cерверная часть ожидает подключения удалённых клиентов из браузера (или запустив бинарный файл клиента Fyne) по http. При появлении удалённого клиента, сервер подключается к агентам на серверах СУДОС по Websocket (наиболее информативный метод контроля соединениями). Агенты, получив подключение извне, начинают периодическую передачу статуса служб на серверную часть программы. Сервер, обработав статусы от агентов, формирует управление GUI по SSE для http-клиентов, тем самым активируя кнопки управления и меняя состояние индикации статусов служб на серверах. На стороне клиентов работает js-сторож активности мышки/клавиатуры, который отключает клиента от серверной программы при бездействии по тайм-ауту. При отсутствии подключённых http-клиентов к серверу, связь с агентами по websocket закрывается до следующего клиента.
